The purpose of this site is to identify and demonstrate html5 using the the primary html5 semantic elements. These currently include <header>, <footer>, <section>, <article>, and <aside>.

The first goal was to make this sample website "fixed" in size. Meaning, no matter how the user resizes the browser view of this site horizontally or vertically, the semantic elements should stay "intack" and viewable. Reference the index.html and style.css files for tested absolute configurations.

Generally speaking, the height of this site is defined with fixed height values for the <section> and <aside> elements. Content within the <header> and <footer> elements can contribute. The width of this site is set with a fixed definition for the <section> block with the <aside> floating to the right. A strict goal of separating "content" from "configuration" is important to understanding how html5 works. The content can and should, at determined times, influence "configuration" to a planned degree.

Also note that moving the <article> section in-between the previous paragraph and this one produced the desired affect of placing this text to the left of and/or under the <article> section depending on the web users horizontal and vertical settings.
